A–Z

ACL (Access Control List)

Список управления доступом, который определяет кто из пользователей системы и на каком уровне может получать доступ к объекту системы.

Riverdoc

Система электронного документооборота ("СЭД", "система"), предназначенная для организации внутреннего и внешнего документооборота, формализации и автоматизации бизнес-процессов, связанных с документооборотом компании, а также совместной работы сотрудников с документами.

Б

Блокировка документа явная

Блокировка документа по команде пользователя, используется для защиты документа от изменений другими пользователями. Автоматическое снятие такой блокировки не предусмотрено, держателю блокировки необходимо выполнить команду Снять блокировку вручную. Система не может продвинуть по жизненному циклу документ, заблокированный пользователем явно. Поэтому мы не рекомендуем устанавливать явную блокировку на продолжительное время.

Блокировка документа неявная

Блокировка документа средствами системы, без участия пользователя. Неявная блокировка, как правило, занимает непродолжительное время (секунды или миллисекунды), необходимое для сохранения изменений.

В

Вид документа

Параметр, позволяющий классифицировать документ по целевому назначению (например, "Заявление на отпуск" или "Служебная записка"). См. также Тип документа.

Г

Группа задач

Задачи одной категории. Объединение задач в группы позволяет автоматически сортировать, а затем просматривать задачи нужной категории в одном списке (например, задачи группы Делаю или задачи группы Поручил).

Д

Делегирование задачи

Переадресация всех обязанностей и ответственности по задаче другому сотруднику.

Ж

Жизненный цикл (ЖЦ)

Совокупность этапов, которые документ проходит от момента создания до завершения исполнения или отправки контрагенту.

З

Задание (job)

Автоматически выполняемое действие, которое позволяет выполнять регламентные работы без участия пользователя, например формировать еженедельные отчёты, создавать резервные копии. От пользователя требуется только задать параметры исполнения и расписание автозапуска с помощью планировщика заданий.

Задача (task)

Действие пользователя или системы, направленное на выполнение одного из этапов документооборота. С помощью задач электронный документ перемещается в СЭД. В Riverdoc задачи делятся на задачи по процессам и задачи по поручениям. Подробнее см. Раздел "Задачи".

Задача новая (или неоткрытая)

Задача, карточку которой пользователь ещё не открывал. Виджет новой задачи выделяется голубым цветом. См. Маркировка и формат записи задач.

Задача просроченная

Задача, которую пользователь не завершил в поставленный срок. Название просроченной задачи выделяется шрифтом красного цвета. См. Маркировка и формат записи задач.

И

История поручения

Журнал движения задач по поручению. История поруения доступна в карточке поручения на вкладке История поручения. Всем исполнителям поручения отображается полная и одинаковая история.

История процесса

Журнал движения задач по процессу. История процесса доступна в карточке любой задачи по этому процессу на вкладке История процесса. Всем исполнителям задач по процессу отображается полная и одинаковая история.

К

Карточка документа

Электронная регистрационная форма, содержащая сводную информацию о документе в структурированном виде: атрибуты, дополнительные файлы, связанные документы, история движения по процессам документооборота, электронные подписи документа, история версий.

Карточка задачи

Электронная форма, в которой содержится содержится сводная информация по задаче: автор, исполнитель(и), инструкция, дата старта, срок исполнения (опционально), вложения, история исполнения, а также команды, необходимые для передвижения и завершения задачи.

Консоль администрирования

Рабочее место администратора системы.

М

Маршрут документа

Схема движения документа в рамках поручения или процесса.

Меню задач

Список видимых групп задач. В этом меню пользователь выбирает группу, задачи которой отображаются в списке справа.Видимость групп можно изменить в пункте Настроить фильтры.

О

Объекты системы

Документы, информация, задачи, которые хранятся в архиве СЭД и участвуют в документообороте.

П

Папка назначения

Папка, в которой будет создан документ или вложенная папка. Папка назначения может быть указана системой автоматически или пользователем вручную. 

Папка по умолчанию

Папка, в которой создаются и хранятся документы определённого вида (например, документы вида "Счёт" создаются и хранятся в директории /Бухгалтерия/Счета/). Папку по умолчанию назначает администратор.

Право доступа

Разрешительное право, которое определяет, какие действия пользователь может выполнять с объектом (документом или папкой).

С

Список задач линейный

Список задач, отсортированный по одному из параметров (название, дата старта, автор и т. д.)

Список задач по приоритету

Список задач, в котором задачи группируются по столбцам в разрезе срочности и важности.,

Т

Тип документа

Параметр, позволяющий классифицировать документ по общему назначению (например, входящие, исходящие и внутренние документы).

Ф

Файловое хранилище

Место хранения содержимого документов в виде файлов. Физически файловое хранилище может располагаться как на сервере, где развернута СЭД, так и на удалённом ресурсе.

Э

Электронный архив

Раздел СЭД, в котором хранятся все электронные документы в структурированном виде.

Электронный документ

Документ, добавленный в систему электронного документооборота. Электронный документ содержит обязательную часть – карточку документа – и может содержать файл документа.
