История процесса

История процесса – журнал движения задач по текущему процессу. История процесса доступна в карточке любой задачи по этому процессу на вкладке История процесса. Всем исполнителям задач по процессу отображается полная и одинаковая история. Синим шрифтом и круглой синей меткой выделяется текущая задача, карточка которой сейчас открыта.

В истории процесса фиксируется следующая информация: 

•Исполнитель – аватар, фамилия и инициалы исполнителя или название группы-исполнителя*.

•Задача – название задачи по процессу.

oистория задачи (отображается, если по задаче были изменения: прикрепляли, открепляли, удаляли документы).

•Старт задачи – дата и время запуска задачи.

•Принята – дата и время, когда сотрудник открыл задачу.
Ячейка пустая до тех пор, пока исполнитель не открыл задачу, и заполняется в тот момент, когда исполнитель открыл задачу (в случае групповой задачи – один из пользователей группы принял задачу в работу).

•Статус – текущий статус задачи. Возможные значения:

oНазначена – задача отправлена пользователю, пользователь ещё не открывал задачу.

oПринята – пользователь получил и открыл задачу (в случае групповой задачи – один из пользователей группы принял задачу в работу).

oДелегирована – пользователь делегировал задачу другому сотруднику, или система автоматически делегировала задачу отсутствующего сотрудника его заместителю (при назначенном замещении). 
Записи о делегированных задач отображаются шрифтом светло-серого цвета, при наведении фокуса на задачу цвет шрифта меняется на чёрный.

oВыполнена – назначенный сотрудник завершил задачу. 
Записи о выполненных задач отображаются шрифтом светло-серого цвета, при наведении фокуса на задачу цвет шрифта меняется на чёрный.

oПрервана – администратор системы прервал выполнение процесса, по которому проходила эта задача.

Примечание. Статусы неоткрытых задач выделяются шрифтом зелёного цвета, например .

•Дата выполнения и комментарии – дата и время, в которое исполнитель выполнил задачу, комментарий исполнителя (если указал).
Ячейка пустая до тех пор, пока исполнитель не выполнил задачу, и заполняется в тот момент, когда исполнитель закрыл задачу. Для делегированных задач эта ячейка автоматически заполняется значением "↓ Делегирована %дата время%".

Дополнительные сведения

•*Как только групповую задачу принял один из участников группы, в истории процесса динамически обновляется информация по задаче и указывается её исполнитель.

•Если задачу выполнил заместитель отсутствующего сотрудника, в столбцах Принята и Статус появляется ссылка замещающим, при нажатии которой отображается, кто выполнил задачу.

•Список задач в истории выводится в том порядке, в каком создавались задачи в пределах одной цепочки. Например при параллельной рассылке задач порядок будет соответствовать порядку перечисления исполнителей при запуске процесса.

Цепочки задач

Цепочка задач – связанные между собой задачи, которые выполняются в рамках одного процесса или поручения. В истории процесса цепочку задач можно определить по вертикальным линиям между аватарами исполнителей.

История задачи по процессу

История задачи по – журнал изменений по задаче с момента начала выполнения задачи до момента окончания.

В истории процесса история по конкретной задаче выводится в виде выпадающего блока при щелчке по ссылке история задачи.

Как посмотреть историю задачи по процессу

Ссылка появляется только для задач, пакеты документов которых изменялись в ходе выполнения задачи (прикрепляли, открепляли, удаляли документы).

В истории задачи фиксируются события:

•документ добавили в пакет документов;

•документ удалили из пакета документов;

•документ удалили из системы.

В истории задачи фиксируется информация:

•дата и время события;

•ФИО пользователя, инициировавшего событие;

•действие с документом (добавлен/откреплён/удалён);

•название пакета;

•название и пиктограмма документа.

История задачи по процессу

В истории задачи по процессу при наведении курсора мыши на документ появляется виджет документа, откуда доступны следующие действия с документом: 

скачать файл документа,

открыть карточку документа,

открыть карточку документа и показать предварительный просмотр.

Примечание. В истории задачи фиксируются только изменения с основными документами, история изменений по связанным документам не фиксируется.
