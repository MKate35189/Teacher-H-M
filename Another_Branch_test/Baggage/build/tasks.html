Раздел "Задачи"

В Riverdoc электронные документы перемещаются по системе с помощью электронных задач. Каждая задача является одним из этапов поручения или процесса и формируется либо при запуске поручения/процесса, либо после выполнения предыдущей задачи.

При получении задачи пользователю отображается параметры задачи: название и инструкция, отправитель, срок исполнения, приложенные документы и др. Полная информация по задаче содержится в карточке задачи.

В Riverdoc задачи делятся на два типа:

•Задачи по поручениям (примеры: "Исполнение", "Контроль", "Утверждение", "Доработка").

•Задачи по процессам (примеры: "Подпишите заявление на отпуск", "Расчёт отпускных").

Поручения преднастроены в системе и логику их работы невозможно изменить, соответственно, карточки задач по поручениям одинаковы для всех компаний. Процессы настраивает администратор системы под конкретные требования компании, соответственно, вид карточки задачи по процессу зависит от системных настроек. Для пользователя при выполнении задач по поручению и задач по процессу принципиальной разницы нет, есть некоторые отличия в карточках таких задач (примеры см. в Карточки задач по поручению и Карточка задачи по процессу).

Структура раздела

В разделе Задачи содержатся все активные задачи пользователя, объединённые по группам:

•Делаю – входящие задачи по процессам и поручениям, исполнителем которых вы являетесь.

•Поручил – задачи "Поручение", автором и/или постановщиком которых вы являетесь.
Завершать задачи "Поручение" вручную не нужно – они завершаются автоматически, когда все исполнители поручения завершили свои задачи, и исчезают из списка Поручил.

•Контролирую – задачи "Контроль" по поручениям, находящиеся у вас на контроле.
Завершать задачи "Контролирую" вручную не нужно – они завершаются автоматически, когда контролёр утвердил задачу "Утверждение".

•Избранные – задачи по процессам и поручениям, которые вы добавили в избранное ().

В подгруппах Срочные и Просроченные дублируются задачи основной группы (Делаю/Поручил/Контролирую), на срок выполнения которых стоит обратить внимание:

oСрочные – задачи, до конца срока выполнения которых осталось менее заданного периода (по умолчанию – три дня). Такие задачи помечаются значком .

oПросроченные – незавершённые задачи, срок выполнения которых уже истёк. Название таких задач выделяется шрифтом красного цвета.

Группировка задач позволяет автоматически сортировать, а затем просматривать задачи нужной категории в одном списке (например, задачи группы Делаю или задачи группы Поручил). Пользователь может настроить список отображаемых групп по своему усмотрению с помощью пункта [11] Настроить фильтры.

Примечание. Одна и та же задача может дублироваться у пользователя в разных группах задач, например в Поручил и Контролирую, если пользователем является постановщиком и контролёром задачи.

Назначение раздела

Раздел Задачи позволяет пользователю:

•запускать процессы;

•создавать поручения;

•выполнять или делегировать входящие задачи;

•контролировать выполнение своих поручений;

•подписывать документы, участвующие в документообороте;

•знакомиться с историей поручений и процессов, задачи по которым поступили пользователю;

•а также создавать документы без перехода в раздел Документы.
