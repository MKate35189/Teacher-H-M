Основные правила создания документов в Riverdoc

▪Создать документ можно из любого раздела системы.

▪В корне раздела Документы объекты может создавать только администратор из Консоли администрирования.

▪На папку, в которой планируете создать объект, должно быть право доступа как минимум "Перезапись версий" (Write).

▪При создании документа в Riverdoc необходимо указать вид документа. Изменить вид документа можно только до первого сохранения карточки документа.

▪Папка назначения может быть указана системой автоматически или пользователем вручную.

▪Если в создаваемом документе есть файл, то название файла используется в качестве названия нового документа. Название можно изменить в любой момент.

▪При создании объекта в папке, которая связана с другими папками, объект автоматически будет создан во всех связанных папках.

▪Riverdoc позволяет загружать файлы любых форматов, ограничение размера загружаемого файла устанавливает администратор системы.

Шаг 1. Выбрать вид документа (обязательно)

Вид документа – параметр, по которому документы классифицируются в системе (например, "Заявление на отпуск" или "Служебная записка"). Справочник видов документов настраивает администратор системы.

На первом этапе создания документа открывается окно Создание документа, в котором содержатся все доступные виды документов (справочник документов).

Справочник видов документов

По умолчанию записи в справочнике выводятся списком, в котором виды следуют в алфавитном порядке от А к Я. Также все записи в справочнике можно вывести в виде дерева ("иерархией"). Переключение между режимами выполняется с помощью кнопок .

Примечание. Структура дерева будет соответствовать структуре видов документов, заведённой администратором в Консоли администрирования.

Чтобы выбрать вид документа, наведите курсор мыши на нужный вид документа, далее нажмите кнопку Выбрать.

Выбор вида документа

Откроется форма для заполнения карточки документа (окно Создание документа).

Шаг 2. Прикрепить файл (необязательно)

Если пользователь создаёт документ с помощью перетаскивания файла, тогда файл автоматически добавится в карточку документа.

Если пользователь создаёт документ с помощью кнопок, тогда файл можно прикрепить в области добавления файла:

Область добавления файла

Область добавления файла при наведении курсора

В обоих случаях файл можно удалить с помощью и добавить другой.

Шаг 3. Проверить или указать папку назначения (обязательно)

Папка назначения может быть указана системой автоматически или пользователем вручную. Выбор и назначение папки зависит от нескольких факторов:

•из какого раздела пользователь создаёт документ;

•настроена ли папка по умолчанию* для выбранного вида документа;

•прав доступа пользователя на папку назначения.

*Папка по умолчанию – папка, в которой создаются и хранятся документы определённого вида (например, документы вида "Счёт" создаются и хранятся в директории /Бухгалтерия/Счета/). Папку по умолчанию назначает администратор.

Шаг 4. Заполнить поля карточки (для некоторых видов документов это необязательно)

Набор полей в карточке документа зависит от выбранного вида документа. Администратор системы настраивает поля для каждого вида документов, для всех видов обязательно будут поля:

•Имя документа* – название документа.
Правила именования объектов в Riverdoc:

oОграничение длины имени: 255 символов.

oИмя документа не может быть пустым.

oЕсли пользователь не ввёл название документа, система автоматически сгенерирует название в соответствии с выбранным видом документа.

•Описание – текстовый комментарий к документу, с помощью которого можно понять суть документа, не открывая его.

•Вид документа* – один из видов документов, заведённых в справочнике документов Riverdoc.

Назначение остальных настраиваемых полей должно быть понятно из названия или предзаполненного текста-подсказки. Если назначение какого-либо поля всё же непонятно, обратитесь к администратору системы.
